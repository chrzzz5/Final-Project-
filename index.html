This project is a matching game, it's a fun way to go by with your time when bored. The game was made with javascript, HTML, and CSS. In order for this game to work, a grid was needed, as well as an array that randomizes where the different objects will be on the grid for the users to guess and match. Just like the stopwatch, a countdown is started with a function that allows for a timer to react once the user starts the game. Not only will a timer be started, but the computer will be able to read the core logic behind a memory game, which is reading if two objects match. Constants are used to make sure users can't click other buttons.

Connect 4 game was a complicated game. Although there isn't an actual animation style that showcases the dropping affect of connect 4, the game still works as usual! Instead of using a grid like the matching game, my fellow classmates and I decided to use cell indexes' to create the grid of connect 4. Not only did that take forever, but in connect 4 you have multiple ways that you can win, so we had to create multiple win conditions, there are horizontal, vertical, and right/left win conditions. We used functions that would constantly update the game to read when it's red or yellows turn, as well as when someone wins the actual game.
